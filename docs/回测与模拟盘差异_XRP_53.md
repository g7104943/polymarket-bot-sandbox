# 回测与模拟盘差异说明（XRP_53 / logs_xrp）

## 现象

- **模拟盘**（report_summary）：222 笔，胜率 49.5%，当前资金 $77.68，几乎归零。
- **22 组合统一回测**（backtest_22_combos_unified_ranking.csv）：XRP_53 最终资金 832366，胜率 54.8%，10410 笔，数据很好。

## 是否「正常」？

模拟盘快归零在**统计上可以发生**（53% 阈值接近随机，连续亏损会放大），但和回测差距巨大，需要从**数据是否泄漏**和**规则是否一致**两方面看。

## 回测是否泄漏了数据？

**结论：22 组合统一回测里，旧模型（含 XRP_53）存在训练/测试重叠，存在泄漏风险。**

### 1. 统一回测的「无泄漏」起点是怎么定的

- `backtest_22_combos_unified.py` 里只用了一个起点：
  - `no_leak_start = get_no_leak_start_date(data_src, None, "BTC_USDT")`
- 该函数按 **GRU 的时序划分**（config 里 train+val 比例）在 **parquet 长度**上算出一个「测试集起始日」。
- 即：**同一起点、同一段 365 天** 给所有 22 个组合用，包括 5 个旧模型（ETH/BTC/XRP）和 8 个 GRU。

### 2. 旧模型（XRP_53）的训练时间范围

- XRP 旧模型路径：`data/models_C/lightgbm_XRP_USDT_15m_20260123_152549/`
- metadata 中：
  - `train_cutoff_date`: **2025-10-25**（训练时 holdout 起点，即训练数据用到 2025-10-25 之前）
  - `holdout_days`: 90  
  即：**模型在训练时已经用过 2025-10-25 之前的数据**。

### 3. 泄漏如何产生

- `get_no_leak_start_date` 是按 **整段 parquet 的 80% 位置** 之类算的（和 GRU 的 train/val/test 一致），通常会在 **2024 或 2025 年某日**。
- 若算出的 `no_leak_start` **早于 2025-10-25**，则回测区间 `[no_leak_start, no_leak_start+365天]` 会**包含模型训练期**。
- 例如：no_leak_start = 2024-08-01，则 365 天到 2025-07-31，**整段都在 XRP 模型的训练数据之内** → 回测是在「见过」的数据上评估，**存在泄漏**。
- GRU 组合用的是同一套「按 parquet 比例切出的测试段」，对 GRU 而言是严格未参与训练的；但对旧模型而言，**没有按各自 `train_cutoff_date` 再截断**，因此旧模型（含 XRP_53）容易在「训练期内」做回测。

### 4. 为何会「回测很好、模拟很差」

- **回测**：若存在上述泄漏，相当于在模型「见过」的行情上评估，表现容易偏乐观（832k、54.8%）。
- **模拟盘**：是真正的样本外、实时数据（2026-01 至今），模型未在训练中见过，表现会回归真实泛化能力；53% 阈值本就接近随机，连续亏损导致资金从 400 跌到 77 在统计上不反常。

## 建议

1. **旧模型回测应使用「各自」的样本外起点**  
   - 对每个旧模型（logs_eth / logs_btc / logs_xrp / logs_xrp_20_80）从对应模型目录读取 `metadata.json` 的 `train_cutoff_date`。  
   - 回测区间应为：**start = max(no_leak_start, model_train_cutoff_date)**，end = start + 365 天（且不超过数据末尾）。  
   - 这样旧模型只在「训练时未用过」的区间上评估，与 GRU 的「无泄漏」含义一致。

2. **重新跑 22 组合统一回测**  
   - 对旧模型按上面方式改 start 后再跑一次。  
   - 若 XRP_53 在**严格样本外**的 365 天里表现明显变差（胜率/资金接近模拟盘量级），可反证此前回测确实含有泄漏。

3. **模拟盘「快归零」本身**  
   - 在未改回测前，可先视为：**样本外表现 + 53% 阈值 + 近期行情** 下的结果；  
   - 若希望模拟更稳健，可考虑提高 XRP 阈值（如 0.55）或降低仓位、加风控。

## 已做修改（backtest_22_combos_unified.py）

- 新增 `_get_old_model_train_cutoff_date(symbol, models_dir)`：从旧模型目录下 `metadata.json` 读取 `train_cutoff_date`，返回 YYYY-MM-DD。
- 对 **type=="old"** 的 5 个组合：回测起点改为 `start_old = max(no_leak_start, train_cutoff_date)`，结束为 `start_old + 365 天`（不超过数据末尾），再调用 `get_trades_old` / `get_trades_old_dual`。GRU 组合仍用原来的 `no_leak_start`～`end_365`。
- 重跑后旧模型会打印「样本外起点 YYYY-MM-DD」便于核对。

## 修改后回测结果（验证泄漏假设）

重跑 `python3 scripts/backtest_22_combos_unified.py --data-src data/raw --out-dir .` 后：

| 组合    | 修改前（同一起点，含泄漏风险） | 修改后（样本外起点 2025-10-25 起） |
|---------|-------------------------------|------------------------------------|
| XRP_53  | 排名 5，832366，54.8%，10410 笔 | **排名 21，19.98，52.6%，2218 笔** |
| BTC_55  | 排名 20，19.61，53.6%          | 排名 20，86.4，53.4%               |
| ETH_92  | 排名 14，130297                | 排名 12，4876                      |

XRP_53 在**严格样本外**的约 3.5 个月（2025-10-25～2026-02-06）内表现与模拟盘一致（资金几乎归零），说明此前高收益确实来自训练/测试重叠，修复后回测与模拟盘对齐。

## 小结

| 项目         | 说明 |
|--------------|------|
| 模拟盘归零   | 在样本外、高波动、阈值偏松下可能发生，不必然表示实现错误。 |
| 回测数据很好 | 22 组合统一回测对**旧模型**未按各自 `train_cutoff_date` 截断，存在**训练/测试重叠**的泄漏风险。 |
| 已修复       | 旧模型回测已改为用各模型 `train_cutoff_date` 后的 365 天；XRP_53 修复后排名 21、资金约 20，与模拟盘量级一致。 |
