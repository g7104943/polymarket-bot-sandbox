# 所有模型与下单/预测逻辑检查报告

**检查范围**：P(UP) 获取方式、confidence 计算、单/双阈值逻辑、11 组合配置一致性。  
**结论**：所有模型与下单/预测逻辑与训练时一致，未发现错误。

---

## 1. 模型与 P(UP) 获取方式

| 模型类型 | 模型目录/来源 | P(UP) 获取方式 | 代码位置 |
|----------|----------------|----------------|----------|
| 旧模型 LightGBM | data/models, data/models_C | 有 `predict_proba` 则 `model.predict_proba(X)[:, 1]`，否则 `model.predict(X)`（Booster 即 P(UP)） | `predictor.predict_one()` L240-244 |
| 旧模型回测 | 同上 | 同上 | `backtest_simulation.py` L293-301 |
| GRU 预测 | experiments/gru_regime_v1/outputs/models_best | `lgb_model.predict_proba(X)[0, 1]` | `prediction_writer_gru.py` L376 |
| GRU 回测 | 同上 | `lgb_model.predict_proba(X)[:, 1]` | `backtest_gru_regime.py` L387, L458 |

**结论**：所有路径均输出真实 P(UP)，与训练时 `y_prob = model.predict(X_va)` 语义一致。data/models 与 data/models_C 当前为 Booster，走 `predict()` 分支，返回值即 P(UP)。

## 2. 预测方向与 confidence 计算

| 端 | 方向判定 | confidence 公式 | 代码位置 |
|----|----------|-----------------|----------|
| predictor | `pred = 1 if prob >= 0.5 else 0` | `conf = prob if pred==1 else (1-prob)` | predictor.py L247, L307；predict L362 |
| prediction_writer 单模型 | 同上（来自 predict_one） | 同上 | prediction_writer.py L278-286 |
| prediction_writer 集成 | 多模型 prob 取平均后 `pred = 1 if prob>=0.5 else 0` | 同上 | prediction_writer.py L251-258 |
| prediction_writer_gru | `pred = 1 if prob>=0.5 else 0` | 同上 | prediction_writer_gru.py L379-382 |
| backtest_simulation | 使用 df 中 pred_up / pred_prob | effective_prob = prob if pred==1 else (1-prob) | backtest_simulation.py L401-402 |
| backtest_gru_regime | 使用 df 中 pred_up / pred_prob | effective_prob = prob if pred==1 else (1-prob) | backtest_gru_regime.py L247 |

**结论**：所有端统一为「预测方向的概率」：UP 时 confidence=P(UP)，DOWN 时 confidence=1-P(UP)，与训练时一致。

## 3. 阈值逻辑（单阈值 / 双阈值）

| 端 | 单阈值 | 双阈值 UP | 双阈值 DOWN | 代码位置 |
|----|--------|-----------|-------------|----------|
| backtest_simulation | effective_prob < prob_threshold → 跳过 | prob < prob_threshold_up → 跳过 | prob >= prob_threshold_down → 跳过 | backtest_simulation.py L404-416 |
| backtest_gru_regime | effective_prob < prob_threshold → 跳过 | （GRU 组合均为单阈值，未用双阈值） | - | backtest_gru_regime.py L247-248 |
| TS thresholdMet | confidence >= PROB_THRESHOLD | confidence >= PROB_THRESHOLD_UP | confidence **>** (1 - PROB_THRESHOLD_DOWN) | prediction_api.ts L52-60 |
| prediction_index | 使用 thresholdMet(p) 过滤后下单 | 同上 | 同上 | prediction_index.ts L274 |

**结论**：单阈值等价于「预测方向概率 ≥ 阈值」；双阈值 UP 为 P(UP)≥up，DOWN 为 P(UP)<down（TS 用 confidence > (1-down) 表达），与训练时 P(UP) 语义一致。

## 5. 下单流程（TypeScript）

1. `prediction_index.ts` 每秒读 `predictions${PREDICTION_SUFFIX}.json`。
2. `parseLocalPredictions(raw.data, TIMEFRAME, ALLOWED_MARKETS)` 按 timeframe 与 ALLOWED_MARKETS 过滤。
3. `toOrder = parsed.filter(p => p.direction != null && thresholdMet(p))`，仅保留达到阈值的预测。
4. `executor.executeRound({ targetPeriodEndTs, predictions: toOrder })` 下单。

**结论**：下单前统一经 `thresholdMet(p)` 过滤，无二次阈值或逻辑分支冲突。

## 6. 检查结论

- **所有模型**：P(UP) 来源正确（predict_proba 或 Booster predict），与训练时一致。
- **所有预测逻辑**：方向 pred = 1 if prob>=0.5 else 0，confidence = prob if pred==1 else (1-prob)，与训练时一致。
- **所有下单/阈值逻辑**：单阈值与双阈值（UP/DOWN）与训练时 P(UP) 语义一致；TS 与 Python 回测规则一致。
- **11 组合**：启动脚本、回测配置、文档一致，未发现错配。

**未发现逻辑错误。** 若某组合实盘/模拟表现与预期不符，可优先排查：数据延迟、滑点与手续费、或该组合对应模型/数据是否与训练时一致。
