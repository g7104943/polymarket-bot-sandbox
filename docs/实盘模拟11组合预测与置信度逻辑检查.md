## 2. Python 端：预测值与置信度

### 2.1 旧模型（6 组合）— `src/python/prediction_writer.py` + `predictor.py`

- **入口**：`predict_with_details()` → `predict_one(model, feats, df)`。
- **概率来源**（已修复）：
  - `predictor.predict_one()` 优先使用 **`model.predict_proba(last)[0, 1]`**（P(UP)），无则退回 `model.predict(last)[0]`。
  - 避免用 `predict()` 的 0/1 当概率，保证双阈值逻辑有效。
- **置信度**：`conf = prob if pred == 1 else (1 - prob)`，即：
  - UP：confidence = P(UP)；
  - DOWN：confidence = P(DOWN) = 1 - P(UP)。
- **写入 JSON**：`direction`（UP/DOWN）、`confidence`（0–1）、`symbol`、`timeframe` 等。

### 2.2 GRU（5 组合）— `src/python/prediction_writer_gru.py`

- **概率来源**：`lgb_model.predict_proba(X)[0, 1]`，始终为真实 P(UP)。
- **置信度**：`conf = prob if pred == 1 else (1 - prob)`，与旧模型一致。
- **写入 JSON**：同上。

### 2.3 结论（Python）

- 新旧模型写入的 **confidence 均为「预测方向的概率」**（UP 时为 P(UP)，DOWN 时为 P(DOWN)）。
- 旧模型已统一用 **predict_proba** 产出概率，与回测及双阈值逻辑一致。

## 3. TypeScript 端：阈值过滤

### 3.1 读取与过滤

- **文件**：`polymarket/src/utils/prediction_api.ts`、`polymarket/src/prediction_index.ts`。
- **流程**：`readLocalPredictionsRaw()` → `parseLocalPredictions(data, TIMEFRAME, ALLOWED_MARKETS)` → 按 `direction` 与 `confidence` 做阈值过滤。

### 3.2 单阈值（9 组合）

- 条件：`PROB_THRESHOLD_UP` 与 `PROB_THRESHOLD_DOWN` 未同时设置。
- 逻辑：`p.confidence >= PREDICTION_ENV.PROB_THRESHOLD`。
- 含义：只做「预测方向上的置信度 ≥ 阈值」的预测，与 Python 的 confidence 语义一致。

### 3.3 双阈值（2 组合：10-80 / 10-90 / 20-80）

- 条件：`PROB_THRESHOLD_UP != null && PROB_THRESHOLD_DOWN != null`。
- 逻辑（与训练时一致，详见 **docs/概率与阈值逻辑.md**）：
  - **UP**：`p.confidence >= PROB_THRESHOLD_UP`（即 P(UP) ≥ up）。
  - **DOWN**：`p.confidence > (1 - PROB_THRESHOLD_DOWN)`（即 P(UP) < down，与训练时 P(UP) 语义一致）。
- 实现位置：`prediction_api.ts` 导出 `thresholdMet(p)`，`prediction_index.ts` 使用该函数过滤。

### 3.4 ALLOWED_MARKETS 过滤

- `parseLocalPredictions` 中：`base = pred.symbol.replace(/\/.*$/, '')`，保留 `set.has(pred.symbol) || set.has(base)` 的预测。
- 启动脚本中每个组合单独设置 ALLOWED_MARKETS（如 ETH、XRP、BTC、SOL），保证每个进程只处理对应币种。

### 3.5 结论（TypeScript）

- 单/双阈值与训练时一致；DOWN 方向使用 `confidence > (1 - PROB_THRESHOLD_DOWN)`，即 P(UP) < down，与训练时 P(UP) 语义一致。
- 阈值逻辑统一在 `prediction_api.ts` 的 `thresholdMet(p)` 中，模拟/实盘共用。
- 每组合通过 env 使用各自阈值，ALLOWED_MARKETS 与 JSON 中的 symbol 匹配正确。

## 6. 回测 0 笔 vs 模拟有交易（如 XRP 20-80）

- **原因**：回测使用「防泄露起始日 + 365 天」的**固定历史窗口**（多为较早时段）；模拟使用 **update_latest 拉取的最新数据**（当前/近期）。同一模型在不同时段预测分布可能不同：在历史 365 天里 XRP 模型可能从未输出 ≥0.8 或 <0.2，在近期则可能有时触发。
- **结论**：回测 0 笔、模拟有笔数属**时段差异**，不是逻辑错误。
- **对比近期**：若想看在「与模拟相近的时段」回测是否也有交易，可加 `--recent-days 90` 跑 11 组合回测（用最近 90 天数据），例如：
  ```bash
  python3 scripts/backtest_gru_regime_report.py --all-11-combos --recent-days 90 --order-price 0.503 --output data/reports/gru_regime_backtest_ranking_11combo.csv
  ```

## 7. 模拟 XRP 20-80 在 1/31 后无交易的可能原因

- **现象**：总汇总报告显示 1/31 16:00:24 最后一笔，2/1、2/2 每日报告均为 0 笔；配置与阈值未改（仍为 UP≥80%, DOWN≥80%）。
- **最可能原因**：**`predict_one()` 改为使用 `predict_proba` 的代码变更**（在 `src/python/predictor.py` 中），若在 1/31 或 2/1 部署/重启预测进程，会导致行为变化：
  - **改之前**：若当时用的是 `model.predict()`，对标准 LightGBM 会返回 **0 或 1**（类别），则 confidence 只会是 0 或 1；双阈值要求 UP≥80%、DOWN≥80%，confidence=1 时都会通过，相当于**几乎每笔预测都会下单**。
  - **改之后**：使用 `model.predict_proba(last)[0, 1]` 得到**真实 P(UP)**，只有 confidence≥80% 或 ≤20%（对应 DOWN≥80%）才下单；若 2/1 起模型输出多在 (0.2, 0.8)，就会**没有交易**。
- **其他可能**：实时更新失败（如 Parquet 报错）导致退回到「仅本地」、数据变少或变旧；或进程重启时环境/配置不一致。可查 `logs/prediction_writer_F.log` 或 XRP 20-80 的 prediction_writer 日志里是否有「实时更新失败」「仅本地」等。
- **建议**：
  1. 若希望 XRP 20-80 在「真实概率」下仍有一定下单频率，可适当放宽双阈值（例如 UP≥70%、DOWN≥70%，即 0.7/0.3），在启动脚本里设 `PROB_THRESHOLD_UP=0.7`、`PROB_THRESHOLD_DOWN=0.3`。
  2. 用 `--recent-days 90` 跑 11 组合回测，看近期 90 天里 XRP 20-80 若用真实概率 + 0.8/0.2 会有多少笔，以确认近期模型输出分布。
