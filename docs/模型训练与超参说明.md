# 模型训练与超参说明

本文说明我们如何训练预测模型、如何做超参调优，以及模型在系统中的作用。

---

## 一、模型的作用

**核心目标**：为 **Polymarket 加密货币 15 分钟涨/跌预测市场** 提供「下一根 K 线涨还是跌」的预测，用于模拟盘或实盘的下单决策。

- **输出**：对每个 15m 周期，模型给出 **P(UP)**（下一根 K 线上涨的概率）；结合阈值（如 0.55、0.92）和可选的分档下注（p1～p4），决定是否下单、下多少。
- **标签定义**（与训练一致）：**二分类** —— 0 = DOWN（下一根跌），1 = UP（下一根涨）；由「下一根 K 线收盘价相对当前收盘的涨跌幅」与阈值（如 0.1%）比较得到，见 `feature_engineering.build_labels` 与 `docs/概率与阈值逻辑.md`。
- **使用端**：预测写入 `polymarket/predictions.json`（或带后缀的 predictions_*.json），由 TypeScript 交易进程读取；满足「置信度 ≥ 阈值」时按配置下单（固定比例或档位 p1～p4），买入价、结算规则与 Polymarket 15m 市场一致（如回测用 0.527、赢 1/P−1、输全损）。

因此：**模型的作用 = 在每根 15m K 线收盘前，预测下一根 15m 是涨是跌，并给出概率，供下游做「是否下单、下多少」的决策。**

---

## 二、两套模型体系

### 1. 旧模型（纯 LightGBM）

- **训练代码**：`src/python/model_trainer.py`（如 `train_one`、`train_with_cv`、`train_per_timeframe`）。
- **数据**：OHLCV K 线（来自 `data/raw/` 的 parquet），按 **交易对 + 时间周期**（15m / 1h / 4h）分别训练**独立模型**。
- **特征**：由 `feature_engineering` 构建 —— KDJ、MACD、RSI、ATR、Range、多周期 mtf_* 等；可选「仅主要特征」或特征 allowlist。
- **标签**：`build_labels` 得到的二分类（下一根 K 线涨/跌），可选涨跌幅阈值（label_threshold_pct）与 lookahead。
- **训练方式**：
  - 时间序列划分（如 80% 训练、20% 验证），或时间序列交叉验证（TimeSeriesSplit）；
  - 可选 **purge/embargo**（训练末尾与验证开头各去掉若干行）减小时序泄漏；
  - 类别不平衡时自动设 `scale_pos_weight`；
  - 可选 **概率校准**（isotonic / sigmoid），校准器随模型一起保存，预测时使用。
- **输出**：模型与特征列表、metadata（含 train_cutoff_date、校准等）保存到 `data/models/`（或指定目录）；月度更新等入口见 `model_improvement.monthly_update`。

### 2. GRU 模型（GRU Encoder + LightGBM）

- **工程目录**：`experiments/gru_regime_v1/`；入口 `run_experiment.sh`（single / grid / random / all 等）。
- **思路**：**不替换原有特征**，而是用 GRU 对价格序列做 **volatility 回归**，得到 **embedding**，将 embedding 作为**新增特征**拼到原有手工特征上，再训练下游 **LightGBM** 做同样的二分类（下一根 K 线涨/跌）。
- **训练流程**：
  1. **GRU Encoder**：用滑窗序列（lookback 根 K 线）预测波动率等目标，训练 GRU；输出 checkpoint（`.pt`）与配置。
  2. **导出 Embedding**：用训练好的 GRU 对全量时间戳做前向，得到每个时间点的 embedding 向量，写入 `outputs/embeddings/<asset>_embeddings.parquet`。
  3. **下游 LightGBM**：将 **原始特征 + embedding 列** 一起作为输入，仍用「下一根涨跌」标签训练二分类；可选 Baseline（仅原始特征）与 +Embedding 对比。
- **超参**：GRU 部分做**网格或随机搜索**（lookback、hidden_size、embedding_dim、lr、batch_size、dropout、weight_decay 等）；下游 LightGBM 使用固定参数（见 `configs/config.json`）。最佳 run 按 val 指标选出，用于导出最终 embedding 与 LightGBM 模型。
- **使用**：模拟/实盘通过 `prediction_writer_gru.py` 等读取 GRU + LightGBM 模型，输出同样为 P(UP) 与方向，与旧模型在「概率与阈值逻辑」上一致（见 `docs/概率与阈值逻辑.md`、`docs/模型与下单逻辑检查报告.md`）。

两套模型**都服务于同一目标**：输出 15m 下一根涨跌的 P(UP)，供统一的下单与回测逻辑使用；差异仅在特征来源（纯手工 vs 手工+GRU embedding）与训练 pipeline。

## 三、超参的两层含义

### 1. 模型超参（训练阶段）

- **旧模型**：LightGBM 参数来自 `config/lightgbm_params.json`（或 optuna 优化版）；训练时的 early_stopping_rounds、val_ratio、purge_embargo、calibration_method 等由脚本或 freqai 配置控制。
- **GRU**：Encoder 的 lookback、hidden_size、embedding_dim、lr、batch_size、dropout 等由 `experiments/gru_regime_v1/configs/config.json` 与网格/随机搜索决定；下游 LightGBM 不参与搜索。

### 2. 交易超参（档位下注、阈值+Δ）

- **含义**：在「已有模型与预测」的前提下，**何时下单、下多少** —— 如「置信度 ≥ 阈值+Δ」「按置信度分档 p1～p4 下注」。
- **调优脚本**：`scripts/hyperparam_tune_13_combos.py`（如 `--val-no-leak`），在**历史无泄漏区间**上回测「阈值+Δ + 分档 p1～p4 + 固定买入价 0.527」等规则，按综合得分（如最差段资金、回撤）选出 **稳健配置**（取各组合 top5 的 Δ、p1～p4 中位数与校准众数），而不是单纯「得分最高那一组」，以减轻过拟合。
- **使用**：稳健配置写入 `hyperparam_tune_13_combos_robust.csv`，模拟/实盘中的「超参组合」由启动脚本读该表，为对应组合设置 阈值+Δ、TIER_P1_PCT～TIER_P4_PCT 等，与回测逻辑一致；见 `docs/超参三个表说明.md`、`模拟交易指令.md`。

因此：**训练超参**决定模型怎么学、学什么特征；**交易超参**决定在已有预测下怎么下单、下多少，二者配合使用。

## 四、训练与使用流程小结

| 阶段       | 旧模型（LightGBM）                    | GRU 模型（Encoder + LightGBM）              |
|------------|----------------------------------------|---------------------------------------------|
| 数据       | OHLCV parquet，按 symbol+timeframe    | 同上 + 滑窗序列                             |
| 特征       | KDJ、MACD、ATR、mtf_* 等              | 同上 + GRU embedding                        |
| 标签       | 下一根 K 线涨/跌（二分类）            | 同上                                        |
| 训练       | model_trainer.train_* / monthly_update| run_experiment.sh → GRU 训 → 导出 emb → LightGBM |
| 输出       | P(UP)、方向、可选校准                 | P(UP)、方向、与旧模型一致                   |
| 交易超参   | 各组合固定阈值或双阈值                | 同上；超参组合用稳健表 Δ、p1～p4           |

**模型的作用**可以概括为：在统一的时间与概率定义下，为 15m 涨跌市场提供**下一根 K 线涨跌概率**，驱动「是否下单、下多少」的决策，并与回测、模拟、实盘共用同一套概率与阈值逻辑。
