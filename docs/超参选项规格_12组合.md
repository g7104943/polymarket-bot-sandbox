# 12 组合超参选项规格（保持/提升胜率，不改模型）

与「模拟盘 12 组合」一致：5 旧模型 + 7 GRU（查看实时日志 1～12）。  
在你说「开始写代码」之前**不构建脚本**，只敲定选项与规则。

---

## 一、第一层：只做置信度边际 Δ（不搜阈值）

- **规则**：只做 **pred_prob ≥ 阈值 + Δ** 的订单；阈值用各组合**现有**阈值，不单独搜阈值。
- **Δ 取值**：**0.01, 0.02, 0.03, 0.04, …, 0.1**（共 10 个点）。
- **实现**：在得到 pred_prob 序列后，仅保留 effective_prob ≥ 阈值+Δ 的 bar 再回测（GRU 过滤后跑 `run_trading_loop`；旧模型同理若支持则做，否则仅对支持边际的组合生效）。

---

## 二、第二层：下注比例 + Δ，可与 p1～p4 配合

- **≥6 万固定 3000**：不变，由 Poly 市场流动性决定，**不断点、不调封顶**。
- **<6 万**：按「当前总资金 × 比例」下注；比例可与 **pred_prob ≥ 阈值+Δ** 配合，并可用 **p1, p2, p3, p4** 作为超参：
  - 按「超过 阈值+Δ 多少」分档（类似 tiered 的档位），每档不同下注比例 p1～p4（%）。
  - 新跑的 8 个 GRU tiered 虽全部失败，若有能用的参数/结构也可以沿用；规则仍是：**过了 6 万总额后固定 3000**，6 万之前按档位比例。
- 即：**第二层 = Δ（0.01～0.1）+ 6 万之前的比例/档位（p1～p4 等）**。

---

## 三、校准：哪种更可能提高胜率与最终价格？

- **作用**：校准不「创造」胜率，而是让**预测概率更接近真实频率**，同一阈值下「刚过线」的 marginal 单会变少，真实胜率更稳、最终价格更可控。
- **两种常用方法**：
  - **Isotonic（等渗回归）**：非参数、单调，能拟合任意单调形状。适合模型概率**系统性偏高/偏低且形状复杂**（如高概率过于乐观）。样本足够时更可能提升「同一阈值下的真实胜率」和稳定性；样本少时易过拟合。
  - **Sigmoid（Platt）**：单参数逻辑曲线，适合**整体偏一个方向**的偏差（如整体偏乐观）。参数少、更稳，数据少时更安全。
- **建议**：
  - 若有**足够验证段**（如 365 天且交易笔数不少）：优先试 **isotonic**，更可能把「高置信度但实际常错」的区间压下去，从而提升胜率观感与最终资金稳定性。
  - 若**数据/笔数偏少**：先用 **sigmoid**，再在验证段上对比「无校准 / isotonic / sigmoid」的胜率与最终资金，选验证段更优且更稳的那档。
- **适用**：当前仅旧模型 5 组合有 calibrator；GRU 路径无校准，若以后给 GRU 加校准则同样逻辑在验证段上对比。

## 四、365 天数据太少：能否不泄漏但数据变多？

- **现状**：无泄漏起始日 `get_no_leak_start_date` 由**当前模型训练时的时序划分**决定（config 里 `train_window_pct` + `val_window_pct`），测试集起点 = 全量 parquet 的 (train_pct + val_pct) 位置。回测用「该日起 365 天」= 无泄漏一年；若 parquet 只到「起点+365 天」，那就只有 365 天无泄漏数据。
- **能否变多？**
  - **在现有模型上**：不能。当前模型已经用固定 train/val 切分训完，**无泄漏区间 = 测试集起点 → parquet 末尾**，长度是固定的；你没法把「训练/验证用过的日期」再算进无泄漏段，否则就是泄漏。
  - **要更多无泄漏数据只有两种**：
    1. **重训时把「测试集」起点前移**：例如用更早的截止日做 train/val（或调小 train+val 占比），让「测试集」从更早的日期开始，这样从测试起点到 parquet 末尾的天数就会变长（如 2 年、3 年）。代价是重训、重新定无泄漏起始日。
    2. **等时间自然增长**：不重训的话，每过一天，无泄漏段就多一天（起点不变、终点变晚），但 365 天之后若你仍只取「起点+365 天」窗口，则不会变长；若改成「起点到 parquet 末尾」则长度会随数据更新变长。
- **结论**：训练阶段一旦定了 train/val 切分，无泄漏区间就定了；想在不泄漏的前提下用更多数据，要么用**更早截止日重训**让 test 起点前移，要么用「起点到今日」的**滚动窗口**随数据更新逐渐变长（不重训时）。

## 五、Δ 与阈值的关系（避免 threshold+Δ>1）

- 入场条件：pred_prob ≥ 阈值 + Δ；pred_prob ∈ [0,1]。若 阈值+Δ > 1，永远达不到，该组合在该 Δ 下 0 笔。
- 约束：**对每个组合只保留 Δ ≤ 1 - 阈值（UP 侧）**。即 Δ ∈ {候选} 且 Δ ≤ 1 - 阈值。
- 候选 Δ 统一：**0.02, 0.04, 0.06, 0.08, 0.10**（粗网格步长 0.02），再按 1-阈值 截断。
- 示例：ETH_92 阈值 0.92 → Δ 最多 0.08（0.02,0.04,0.06,0.08）；BTC_55 / GRU 0.52 等 → 0.02～0.10 全用；XRP_20_80 的 UP 侧 0.8 → Δ≤0.2，0.02～0.10 全用；DOWN 侧 0.2 用 pred_prob ≤ 0.2−Δ，Δ 0.02～0.10 都合法。
- 实现：对每个组合、每个候选 Δ，若 该组合 UP 阈值+Δ > 1 则跳过该 Δ；否则正常做「方案 A」分档和 p1～p4。

## 六、方案 A：档位与校准

- **方案 A**：按「超出阈值的量」分 4 档，每档一个比例 p1～p4（<6 万时 bet = 资金×p_i%，≥6 万固定 3000）。
- **XRP_20_80**：同样方案 A；校准当超参一起搜（无 / isotonic / sigmoid），在 20～80 上对比。
- **超参**：(Δ, p1, p2, p3, p4, 校准)，在无泄漏验证段上选表现最好（或前几名）。

## 七、粗网格 + 2 段验证 + 每组合 top 5（具体参数与流程）

**数据**
- 用：无泄漏起始日 → 取 365 天（或 parquet 末尾，取 min(365, 实际天数)）。**不能**用无泄漏起点之前的数据选参。
- 2 段切分：总长 N 天时，段1 = 前 floor(N/2) 天，段2 = 后 N - floor(N/2) 天。例如 365 天 → 段1 182 天、段2 183 天；若 500 天 → 250 + 250。

**粗网格**
- **Δ**：0.02, 0.04, 0.06, 0.08, 0.10；对每个组合只保留 Δ ≤ 1 - 该组合 UP 阈值。
- **p1～p4**：每档只取 **2, 4, 6, 8, 10**（5 档）→ 5^4 = 625 组。
- **校准**：3 档（无 / isotonic / sigmoid）。旧模型与 XRP_20_80 都搜；GRU 若无校准实现则只输出「无」。

**聚合方式（二选一或都算）**
- **平均**：score = (段1最终资金 + 段2最终资金) / 2。
- **最差段**：score = min(段1最终资金, 段2最终资金)，逼着选「两段都不差」的配置。  
实现时可先做「最差段」排序，输出里同时给出两段资金和平均，便于对比。

**排序与 top 5**
- 每个组合内：按 score 降序排序，取 **前 5 名**（top 5）。
- 可选：对 top 5 的 Δ、p1～p4 取中位数、校准取众数，得到「稳健配置」，再在整段验证上跑一次单独一行。

**输出 CSV 列**
- 组合, 组内排名, Δ, p1, p2, p3, p4, 校准, 得分, 段1最终资金, 段2最终资金, 段1胜率%, 段2胜率%, 段1最大回撤%, 段2最大回撤%, 段1笔数, 段2笔数  
（得分 = 平均 或 最差段，由聚合方式定；排序按 组合 再按 组内排名 1～5）

**流程究竟怎么做（步骤）**
1. 算无泄漏起始日，取验证区间 [start, start+365]（或到 parquet 末），按上面规则切成段1、段2。
2. 对 12 组合逐个：
   - 取该组合阈值（单边 UP 或双阈值 UP/DOWN），得到该组合允许的 Δ 列表（0.02～0.10 且 Δ≤1-阈值）。
   - 对每个 (Δ, p1,p2,p3,p4, 校准)：
     - 段1 回测（方案 A：pred_prob≥阈值+Δ + 四档 p1～p4，≥6万 3000）→ 段1最终资金、胜率、回撤、笔数；
     - 段2 回测 同上 → 段2最终资金、胜率、回撤、笔数；
     - score = 平均 或 min(段1资金, 段2资金)。
   - 按 score 降序取 top 5，写入 CSV 行（组合、组内排名 1～5、Δ、p1～p4、校准、得分、段1/段2 各项）。
3. 可选：每组合 top 5 算中位数/众数得稳健配置，全验证段跑一次，追加一行或单独表。
4. 数据始终只用「无泄漏起点之后」这一段；多段验证 = 这一段内再切段，不用更早历史。
