# 概率与阈值逻辑（训练 = 模拟 = 实盘）

**模拟/实盘必须与训练时同一套概率定义和阈值规则，避免逻辑错误。**

---

## 1. 训练时的定义（唯一标准）

- **标签**（`feature_engineering.build_labels`）：0=DOWN，1=UP；二分类下「下一根 K 线涨跌」。
- **模型**：LightGBM objective=binary，学习的是 **P(class=1) = P(UP)**；验证时 `y_prob = model.predict(X_va)`，`y_pred = (y_prob >= 0.5).astype(int)`。
- **推理时必须一致**：使用同一 **prob = P(UP)**，方向 `pred = 1 if prob >= 0.5 else 0`；下单时的阈值比较也基于该 prob。

---

## 2. 概率与置信度（与训练时一致）

- **prob（P(UP)）**：模型输出的「下一根 K 线上涨」的概率，与训练时模型学的一致。
  - 旧模型（LightGBM Booster）：`model.predict(X)` 即 P(UP)。
  - 旧模型（LGBMClassifier）：`model.predict_proba(X)[:, 1]`。
  - GRU：`lgb_model.predict_proba(X)[0, 1]`。
- **confidence（置信度）**：**预测方向上的概率**，用于与阈值比较。
  - 公式：`confidence = prob if direction==UP else (1 - prob)`。
  - 即：UP 时 confidence = P(UP)；DOWN 时 confidence = P(DOWN) = 1 - P(UP)。

---

## 3. 阈值规则（与训练时一致）

### 3.1 单阈值（PROB_THRESHOLD）

- 仅当 **confidence ≥ PROB_THRESHOLD** 时下单（即「预测方向的概率」达到阈值）。

### 3.2 双阈值（PROB_THRESHOLD_UP + PROB_THRESHOLD_DOWN）

- **UP**：仅当 **confidence ≥ PROB_THRESHOLD_UP** 时下单（此时 confidence = P(UP)）。
- **DOWN**：仅当 **confidence > (1 - PROB_THRESHOLD_DOWN)** 时下单（此时 confidence = P(DOWN)，等价于 P(UP) < PROB_THRESHOLD_DOWN）。
- 中间区间（prob 在 [down, up) 之间）：不下单。

**与单阈值的区别**：单阈值用**一个数**（confidence ≥ 某值）管两个方向；双阈值用**两个数**（UP 一个、DOWN 一个），且数值通常更高，中间不下单。两套策略是**不同进程、各自读自己的阈值**，同一根 K 线若 confidence 同时满足两边的阈值，两个进程都可能下单，这是预期行为。

---

## 4. 各端实现位置

| 端 | 文件 | 说明 |
|----|------|------|
| Python 概率 | `src/python/predictor.py` `predict_one()` | 优先 predict_proba，无则 predict()，返回 (pred, prob) |
| Python 置信度 | `prediction_writer.py` / `prediction_writer_gru.py` | conf = prob if pred==1 else (1-prob)，写入 JSON |
| Python 回测 | `scripts/backtest_simulation.py` | 与训练时一致：effective_prob、双阈值 UP prob≥up / DOWN prob<down |
| TS 阈值 | `polymarket/src/utils/prediction_api.ts` `thresholdMet()` | 单/双阈值过滤，DOWN 用 confidence > (1 - PROB_THRESHOLD_DOWN) |
| TS 下单 | `polymarket/src/prediction_index.ts` | 使用 prediction_api.thresholdMet 过滤后下单 |
