# 批量回测脚本使用说明

## 功能说明

`backtest_batch_models.py` 脚本用于批量回测8个模型目录，每个模型交叉使用2个参数文件，共16个组合。

### 主要特性

1. **交叉测试**：8个模型目录 × 2个参数文件 = 16个组合
2. **只回测15m**：所有币种的15m周期
3. **自动生成Excel报告**：按排名排序，标注组合名称
4. **可配置置信度阈值**：每个模型可单独设置阈值
5. **符合Polymarket规则**：资金计算逻辑已修正

## 8个模型目录

根据文档，8个模型目录为：
- `data/models`
- `data/models_A`
- `data/models_B`
- `data/models_C`
- `data/models_v4`
- `data/models_A_v4`
- `data/models_B_old`
- `data/models_C_old`

## 2个参数文件

- `config/lightgbm_params_optuna.json`
- `config/lightgbm_params_optuna_v4.json`

## 使用方法

### 基本用法

```bash
# 使用默认参数（初始资金400U，回测90天）
python scripts/backtest_batch_models.py
```

### 自定义参数

```bash
# 指定初始资金和回测天数
python scripts/backtest_batch_models.py --initial-capital 400 --test-days 90

# 指定并行进程数（加快速度）
python scripts/backtest_batch_models.py --n-jobs 4

# 指定输出Excel路径
python scripts/backtest_batch_models.py --output-excel data/reports/my_backtest.xlsx
```
python3 scripts/backtest_batch_models.py \
    --initial-capital 400 \
    --test-days 120 \
    --n-jobs 4
### 完整参数列表
python3 scripts/backtest_batch_models.py --initial-capital 400 --test-days 120 --threshold-range --n-jobs 2
./快速清理缓存.sh
./终止后台Python进程.sh
```bash
python scripts/backtest_batch_models.py \
    --initial-capital 400 \
    --test-days 90 \
    --prob-threshold 0.92 \

```

## 置信度阈值配置

### 配置文件位置

`config/backtest_thresholds.json`

### 配置格式

```json
{
  "model_thresholds": {
    "models": 0.92,
    "models_A": 0.92,
    "models_B": 0.85,
    "models_C": 0.92,
    "models_v4": 0.92,
    "models_A_v4": 0.92,
    "models_B_old": 0.92,
    "models_C_old": 0.92
  },
  "default_threshold": 0.92
}
```

### 修改方法

1. 打开 `config/backtest_thresholds.json`
2. 在 `model_thresholds` 中修改对应模型的阈值（0-1之间）
3. 例如：将 `models_B` 改为 `0.85`，则该模型只在置信度>=85%时下注

## Excel报告说明

### 报告位置

默认保存在：`data/reports/batch_backtest_YYYYMMDD_HHMMSS.xlsx`

### 报告内容

#### 汇总排名表

包含以下列：
- **排名**：按总体胜率从高到低排序
- **组合名称**：例如 `models_B+optuna_v4`
- **模型目录**：模型所在目录
- **参数文件**：使用的参数文件
- **置信度阈值**：该模型使用的置信度阈值
- **总交易次数**：所有币种15m的总交易次数
- **总胜场/总败场**：胜场和败场数
- **总体胜率%**：总胜场/总交易次数
- **平均盈亏%**：按交易次数加权的平均盈亏百分比
- **最终资金总和**：所有币种最终资金的总和
- **最大回撤%**：所有币种中的最大回撤
- **状态**：成功或错误信息

#### 详细结果表

每个币种的详细回测结果：
- **组合名称**：组合标识
- **币种**：BTC/ETH/SOL/XRP
- **周期**：15m
- **交易次数**：该币种的交易次数
- **胜场/败场**：胜场和败场数
- **胜率%**：该币种的胜率
- **最终资金**：该币种的最终资金
- **盈亏%**：该币种的盈亏百分比
- **最大回撤%**：该币种的最大回撤

## 资金计算逻辑（已修正）

### Polymarket规则

1. **买入时**：
   - 以50c买入一份合约
   - 合约数 = bet_amount / 0.5
   - 先扣除：`capital -= bet_amount + fee`（投入和手续费）

2. **赢了**：
   - 合约价值1 USDC，所以得到 `bet_amount * 2`（投入翻倍）
   - 净赚 = `bet_amount * 2 - bet_amount - fee = bet_amount - fee`
   - `capital += bet_amount * 2`

3. **输了**：
   - 合约价值0 USDC，所以得到0
   - 净亏 = `0 - bet_amount - fee = -bet_amount - fee`
   - `capital += 0`（已经在买入时扣除了）

### 验证

- ✅ 买入时先扣钱
- ✅ 赢了得到2倍投入
- ✅ 输了得到0
- ✅ 手续费在买入时扣除

## Excel报告说明

### 排名表（第一个工作表）

包含所有组合的所有币种结果，按胜率%从高到低排序。

**列说明**：

1. **排名**：按胜率%排序的排名
2. **币种**：交易对（BTC、ETH、SOL、XRP等）
3. **周期**：时间周期（15m）
4. **交易次数**：总交易次数
5. **胜场**：盈利的交易次数
6. **败场**：亏损的交易次数
7. **胜率%**：胜场 / 交易次数 × 100%（越高越好）
8. **最终资金**：回测结束时的资金（USDC）
9. **盈亏%**：(最终资金 - 初始资金) / 初始资金 × 100%（正数表示盈利）
10. **最大回撤%**：从权益曲线历史高点的最大跌幅（越低越好）
11. **最低/初始%**：曾到达的最低资金 / 初始资金 × 100%（**越高越好**）
    - **100%**：资金从未低于初始资金（最安全）
    - **50%**：曾跌到初始资金的一半（中等风险）
    - **接近0%**：曾几乎归零（风险极高，需谨慎）
12. **组合**：模型目录 + 参数文件的组合名称

### 指标解读

#### 最低/初始%（风险指标）

**含义**：在整个回测期间，资金曾到达的最低点相对于初始资金的比例。

**判断标准**：
- **≥ 90%**：非常安全，资金波动很小
- **70-90%**：较安全，有适度波动
- **50-70%**：中等风险，资金曾明显下降
- **30-50%**：高风险，资金曾大幅下降
- **< 30%**：极高风险，接近破产边缘

**示例**：
- 初始资金：400 USDC
- 最低资金：200 USDC
- 最低/初始% = 200 / 400 × 100% = **50%**
- 说明：资金曾跌到初始资金的一半，风险较高

**为什么越高越好？**
- 这个指标反映策略的**风险控制能力**
- 即使最终盈利，如果过程中资金曾大幅下降，说明策略不够稳健
- 在真实交易中，资金大幅下降可能导致心理压力或被迫止损

#### 最大回撤%（风险指标）

**含义**：从权益曲线历史最高点到最低点的最大跌幅。

**判断标准**：
- **< 20%**：回撤控制良好
- **20-40%**：中等回撤
- **> 40%**：回撤较大，需注意风险

**与最低/初始%的区别**：
- **最大回撤%**：从最高点（可能是盈利后的高点）到最低点的跌幅
- **最低/初始%**：最低点相对于初始资金的比例
- 例如：初始400 → 涨到800 → 跌到200
  - 最大回撤% = (800-200)/800 = 75%（从高点回撤）
  - 最低/初始% = 200/400 = 50%（最低点相对初始资金）

## 注意事项

1. **模型目录必须存在**：如果某个模型目录不存在，会在报告中标记为错误
2. **只回测15m**：脚本会自动过滤，只回测15m周期的所有币种
3. **并行处理**：使用 `--n-jobs` 可以加快回测速度，但会占用更多CPU
4. **Excel依赖**：需要安装 `openpyxl`：`pip install openpyxl`
5. **进程清理**：如果按Ctrl+C停止，脚本会尝试优雅关闭所有子进程；如果仍有后台进程，运行 `./终止后台Python进程.sh`
